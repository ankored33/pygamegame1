# 開発ログについて

このディレクトリには、日付ごとの開発履歴が記録されています。

## 目的
- 異なるAIや開発者が引き継いでも、開発の経緯を理解できる
- 実装した機能、技術的な判断、既知の問題を記録
- 将来の拡張ポイントを明確化

## ファイル形式
- `YYYY-MM-DD.md`: 各日の開発ログ
- Markdown形式で記述

## 記録内容
1. **実装内容**: その日に実装した機能
2. **技術的な詳細**: 新規ファイル、変更ファイル
3. **設計判断**: なぜその実装を選んだか
4. **次のステップ**: 将来の拡張ポイント
5. **動作確認**: テスト結果
6. **既知の問題**: バグや未解決の課題
7. **メモ**: その他の重要な情報

## 使い方
- 開発開始時: 前回のログを確認
- 開発終了時: その日の作業を記録
- 引き継ぎ時: 最新のログから読む
